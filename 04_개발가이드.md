## 핵심 아키텍처 패턴

### 1. 데이터 흐름
```
gon-services (Railway/FastAPI)
    ↓ Primary
getAllLottoData() ← dataFetcher.ts (ISR revalidate)
    ↓ Fallback
realLottoData.ts (정적 백업)
    ↓ Fallback
smok95 API → 동행복권 API
```

### 2. Supabase 타입 안전 패턴

**중요**: Row 타입은 반드시 `type` alias 사용 (`interface` 금지)

```typescript
// database.ts - 올바른 패턴
export type WinningStore = {
  id: number;
  round: number;
  // ...
};

// 틀린 패턴 (Supabase v2에서 타입 추론 실패)
export interface WinningStore {  // ← interface 사용 금지
  id: number;
}
```

모든 테이블에 `Relationships: []` 필수:
```typescript
export interface Database {
  public: {
    Tables: {
      winning_stores: {
        Row: WinningStore;
        Insert: WinningStoreInsert;
        Update: Partial<WinningStoreInsert>;
        Relationships: [];  // ← 필수
      };
    };
  };
}
```

### 3. Window 전역 타입 확장

`src/types/global.d.ts`에서 선언:
```typescript
interface Window {
  __trackAction?: (action: string) => void;
  gtag?: (...args: unknown[]) => void;
  recordUserActivity?: (action: string) => void;
}
```

사용 시 `(window as any)` 금지, 직접 접근:
```typescript
window.__trackAction?.('some_action');
```

### 4. 캐시 무효화 패턴

```typescript
import { invalidateAllDataCache, registerCacheInvalidationCallback } from '@/lib/dataFetcher';

// 다른 모듈의 캐시를 연동 등록
registerCacheInvalidationCallback(() => {
  myModuleCache.clear();
});

// 전체 캐시 무효화 (cron에서 호출)
invalidateAllDataCache();
```

### 5. 인증 패턴

```typescript
import { getAuthFromRequest } from '@/lib/auth';

export async function GET(request: NextRequest) {
  const auth = getAuthFromRequest(request);
  if (!auth) {
    return NextResponse.json({ success: false, error: '로그인이 필요합니다.' }, { status: 401 });
  }
  // auth.userId 사용
}
```

## 코드 규칙

1. `as any` 사용 금지 — 타입을 정확히 정의할 것
2. `console.log` 프로덕션 금지 — `console.error`만 에러 핸들러에서 사용
3. 파일당 400줄 이내
4. Supabase 쿼리는 `getServiceSupabase()` 사용 (서버), `getPublicSupabase()` (클라이언트)
5. 모든 API 응답: `{ success: boolean, error?: string, ...data }`
